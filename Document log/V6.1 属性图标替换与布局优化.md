## 二、防守弱点统计优化

### 问题

防守弱点统计区域的数字气泡与图标不贴合，图标尺寸偏小。

### 修改

| 项目 | 修改前 | 修改后 |
|------|--------|--------|
| 图标尺寸 | `w-[117px] h-[29px]`（PNG 精灵图） | `w-[38px] h-[38px]`（SVG 正方形） |
| 气泡定位 | `absolute -top-1.5 -right-1.5` | `absolute -top-1 -right-1` |
| 容器 | `relative` | `relative inline-block` |
| 气泡尺寸 | `w-5 h-5` / `w-4 h-4` | 统一 `w-4 h-4` |
| 气泡字号 | `text-[10px]` / `text-[8px]` | 统一 `text-[8px]` |

修改位置：`public/index.html:2266-2273`，`pokemon_battle_assistant.html:2276-2283`

## 五、六项属性种族值显示修复（V6.2）

### 问题

宝可梦卡片上六项属性数值（HP、A、D、SA、SD、S）中，只有速度 S 显示正确，其他五项均显示为 80。

### 根因

`handlePokemonNameChange` 函数中，只有 `pokemon.speed` 从 API 返回的 `data.baseStats` 赋值，其余五项属性（`hp`、`attack`、`defense`、`spAttack`、`spDefense`）未被设置。渲染时通过 `pk[key] || 80` 回退到默认值 80。

### 修改

在 `handlePokemonNameChange` 函数中补全六项属性的赋值：

```js
pokemon.hp = data.baseStats.hp;
pokemon.attack = data.baseStats.attack;
pokemon.defense = data.baseStats.defense;
pokemon.spAttack = data.baseStats.spAttack;
pokemon.spDefense = data.baseStats.spDefense;
pokemon.speed = data.baseStats.speed;
```

修改位置：`public/index.html`（原 2726 行）、`pokemon_battle_assistant.html`（原 2741 行）

注意：后续的 `pokemon.hp = 100`（当前 HP 满血重置）保持不变，不影响属性条显示的种族值。
