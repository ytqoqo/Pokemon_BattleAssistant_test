> **分辨率：** 1920×1080（固定）

## 详细脚本

### 1. 开场 & 全局概览 (20s)

**画面：** 浏览器打开 `pokemon_battle_assistant.html`，显示完整对战界面

**操作：**
- 不要做任何操作，静止 2-3 秒让观众看清全貌
- 缓慢滚动展示：上方环境监控 → 中间双方队伍 → 下方速度排名和弱点分析

**台词参考：**
> "这是宝可梦对战助手 V4.3.0，为 VGC 2026 Regulation G 规则设计的辅助决策工具。可以快速追踪双方队伍状态、计算行动顺位、分析属性克制关系。"

---

### 2. 队伍方案管理 (40s)

**画面：** 聚焦左侧下方"队伍方案"区域

**操作：**
1. 点击 **+ 新增队伍方案**，创建一个空白方案（默认6个空槽位）
2. 点击方案名称文字，实时修改名称（如改为 "雨天队"）
3. 点击 **保存到方案**，将当前我方队伍（苍响/咆哮虎/多龙巴鲁托等）保存
4. 点击另一个已有方案，演示 **替换队伍** 功能，左侧队伍瞬间切换

**台词参考：**
> "队伍方案支持本地持久化。可以新建方案、实时修改名称、保存当前队伍，一键切换不同方案，方便在多套队伍之间快速切换。"

**视觉亮点：**
- 方案颜色标识条清晰区分不同方案
- 名称点击即改，失焦自动保存
- 替换队伍时 UI 即时刷新

---

### 3. 速度排名系统 (50s)

**画面：** 聚焦下方速度排名面板（Speed Tracker）

**操作：**
1. 初始状态展示12只宝可梦的速度排名（蓝色=我方，红色=敌方）
2. 修改我方某只宝可梦的速度数值（如苍响从 220 改为 300）
3. 观察排名自动重排
4. 给某只宝可梦添加 **讲究围巾**（演示道具对速度的影响）
   - 点击道具名称 → 打开道具选择器 → 切换到"速度影响"分类 → 选择讲究围巾
   - 速度排名中该宝可梦琥珀色高亮显示修正后速度
5. 给某只宝可梦设置 **麻痹** 状态
   - 点击状态按钮切换到麻痹
   - 速度减半，排名下降

**台词参考：**
> "速度排名系统实时追踪双方12只宝可梦的行动顺位。道具如讲究围巾提供1.5倍加速，麻痹状态减半速度，修正后的数值会高亮显示。"

**视觉亮点：**
- 蓝/红色标识条一眼区分敌我
- 琥珀色高亮速度修正值
- 麻痹状态标签 + 速度排名联动

---

### 4. 弱点分析 (50s)

**画面：** 聚焦弱点分析面板

**操作：**
1. 切换到 **粗略模式**（如果当前是详细模式）
   - 点击模式切换按钮
   - 队伍卡片变紧凑，弱点倍率突出显示
2. 观察弱点分析面板：Top 5 弱点属性，彩色圆点 + 计数
3. 指出某个高亮弱点（如 ≥3次的红色标记）
4. 回到详细模式，点击某个宝可梦卡片，展示单只宝可梦的弱点计算
5. 以 **苍响 (妖精/钢)** 为例：
   - 旧算法会误判 5 个弱点
   - 新算法正确判为仅 1 个弱点（地面 2x）
   - 火/格斗被钢属性抗性抵消，毒被钢属性免疫抵消

**台词参考：**
> "弱点分析统计全队对18种攻击属性的弱点分布。V4.3.0 重写了计算逻辑，正确处理双属性的抗性抵消和免疫，不再简单累加。比如苍响妖精钢属性，实际只有地面一个弱点。"

**视觉亮点：**
- 粗略模式下弱点倍率（2x/4x）醒目显示
- 弱点 ≥3次红色标记
- 属性分布彩色圆点直观展示

---

### 5. 道具系统 & 属性克制计算 (40s)

**画面：** 道具选择器 + 属性克制展示

**操作：**
1. 点击某只宝可梦的 **道具名称**，打开道具选择器
   - 展示顶部分类 Tab（全部/速度影响/树果/讲究/...）
   - 展示搜索框，输入关键词过滤
   - 展示4列网格，每个道具显示精灵图 + 中文名
2. 选择 **生命玉** 或其他道具，确认设置成功
3. 点击道具图标，切换 **消耗状态**（图标变半透明）
4. 回到弱点面板，快速展示属性克制图标（PokeAPI 朱/紫风格精灵图）

**台词参考：**
> "道具系统包含约110个对战道具，支持分类浏览和搜索。道具消耗后速度加成消失，图标半透明提示。属性图标全部采用 PokeAPI 第九世代风格。"

**视觉亮点：**
- 道具选择器分类清晰，精灵图丰富
- 消耗状态 opacity:30% 视觉反馈
- PokeAPI 属性精灵图替代图标

---

### 6. 收尾 (10s)

**画面：** 回到完整界面，静止 2-3 秒

**台词参考：**
> "以上就是宝可梦对战助手 V4.3.0 的核心功能演示。"

## 录制前准备清单

- [ ] 启动后端：`npm start` 或 `npm run dev`
- [ ] 浏览器打开 `pokemon_battle_assistant.html`
- [ ] 确认图片正常加载（PokeAPI 外网可达）
- [ ] 清空 localStorage 中的旧方案（或准备2-3个预设方案）

- [ ] 准备录屏软件（OBS/系统自带），设置 1920×1080
